# 01 - 硬件等级定义（L0 → L3）

> **Simo 的硬件演进不是"一步到位"，而是"渐进式升级"。**

---

## 🎯 设计哲学

### 为什么要分等级？

1. **降低入门门槛**：L0 只需一台电脑就能体验
2. **验证核心价值**：先验证"人格"，再加"身体"
3. **控制风险**：每一级都是可用的产品
4. **灵活适配**：不同用户可以停在不同等级

### 核心原则

> **每一级都必须是"完整的体验"，而不是"半成品"。**

---

## 📊 等级定义

### L0 - 纯软件（当前默认）

**定位**：验证 Simo 人格和对话能力

```
┌─────────────────────────────────┐
│  浏览器 / 手机 / 平板           │
│  ┌───────────────────────────┐ │
│  │  Simo 对话界面            │ │
│  │  - 文字对话               │ │
│  │  - 语音合成               │ │
│  │  - 记忆系统               │ │
│  └───────────────────────────┘ │
└─────────────────────────────────┘
```

**硬件需求**：
- 任意能运行浏览器的设备
- 可选：麦克风（语音输入）

**能力**：
| 功能 | 状态 |
|------|------|
| 文字对话 | ✅ |
| 语音合成 | ✅ |
| 记忆系统 | ✅ |
| 多模型支持 | ✅ |

**适用场景**：
- 开发调试
- 体验 Simo 人格
- 无硬件用户

---

### L1 - 定点存在

**定位**：Simo 有了"固定的身体"，但不会移动

```
┌─────────────────────────────────┐
│  外接屏幕（显示表情/状态）       │
│  ┌───────────────────────────┐ │
│  │  👀 Simo 表情             │ │
│  │  💬 对话气泡              │ │
│  └───────────────────────────┘ │
│                                 │
│  🔊 扬声器                      │
│  🎤 麦克风阵列                  │
└─────────────────────────────────┘
```

**硬件需求**：
| 组件 | 规格 | 用途 |
|------|------|------|
| 显示屏 | 5-7寸 LCD/OLED | 表情、状态 |
| 扬声器 | 3W 小音箱 | 语音输出 |
| 麦克风 | USB 麦克风阵列 | 语音输入 |
| 主机 | 树莓派 / 旧手机 | 计算 |

**能力**：
| 功能 | 状态 |
|------|------|
| L0 全部功能 | ✅ |
| 表情显示 | 🔜 |
| 语音唤醒 | 🔜 |
| 本地语音识别 | 🔜 |

**适用场景**：
- 桌面陪伴
- 床头助手
- 厨房助手

---

### L2 - 简单移动 ← **当前进度**

**定位**：Simo 可以"走动"了

```
┌─────────────────────────────────┐
│  [ITX/x86 主机]                 │
│         │ USB                   │
│         ↓                       │
│  [STM32 控制板]                 │
│         │                       │
│    ┌────┼────┬────┐             │
│    ↓    ↓    ↓    ↓             │
│  电机  舵机  超声波  红外        │
│                                 │
│  🚗 四轮底盘                    │
└─────────────────────────────────┘
```

**硬件需求**：
| 组件 | 规格 | 用途 | 状态 |
|------|------|------|------|
| 主控板 | STM32F103C8T6 | 实时控制 | ✅ |
| 上位机 | ITX/x86 | 计算核心 | 规划中 |
| 电机 | TT马达 ×4 | 移动 | ✅ |
| 电机驱动 | L298N | 电机控制 | ✅ |
| 舵机 | SG90 | 超声波扫描 | ✅ |
| 超声波 | HC-SR04 | 测距 | ✅ |
| 红外 | 双路 | 近距避障 | ✅ |

**能力**：
| 功能 | 状态 |
|------|------|
| L1 全部功能 | ✅ |
| 前进/后退/转向 | ✅ |
| 超声波避障 | ✅ |
| 自主探索 | ✅ |
| 语音控制运动 | ✅ |
| 跟随 | 🔜 |

**适用场景**：
- 房间内移动
- 简单跟随
- 避障探索

---

### L3 - 智能交互

**定位**：Simo 成为真正的"家庭成员"

```
┌─────────────────────────────────┐
│  [视觉系统]                     │
│  - 摄像头                       │
│  - 人脸识别                     │
│  - 物体检测                     │
│                                 │
│  [情感系统]                     │
│  - 表情动画                     │
│  - 情绪状态                     │
│  - 主动行为                     │
│                                 │
│  [空间系统]                     │
│  - SLAM 建图                    │
│  - 路径规划                     │
│  - 空间记忆                     │
└─────────────────────────────────┘
```

**硬件需求**：
| 组件 | 规格 | 用途 |
|------|------|------|
| 摄像头 | USB/CSI | 视觉输入 |
| 深度相机 | RealSense/OAK | 3D感知 |
| 显示屏 | 圆形 LCD | 表情 |
| 激光雷达 | 可选 | SLAM |

**能力**：
| 功能 | 状态 |
|------|------|
| L2 全部功能 | ✅ |
| 人脸识别 | 🔜 |
| 情感表达 | 🔜 |
| 主动行为 | 🔜 |
| 空间记忆 | 🔜 |
| 路径规划 | 🔜 |
